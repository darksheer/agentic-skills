## 2024-05-14 - Bash JSON Parsing Optimization
**Learning:** In bash scripts, repeatedly parsing the same JSON payload using `jq` creates a significant performance overhead, especially for large responses like the sessions list from Jules API. `jq` launches a new process each time. In `tests/run_all.sh` inside `run_pipeline()`, multiple `jq` calls are made on the `$sessions` JSON payload.
**Action:** We can combine multiple `jq` extractions into a single `jq` call, outputting them in a format bash can easily read (like a space-separated string or an array), reducing the number of `jq` processes spawned and making the code more efficient. Or at the very least, gracefully handle empty or errored JSONs to avoid `Cannot iterate over null` which actually causes test failures when testing locally without a real API key.
