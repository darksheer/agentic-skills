# PR Report Template

Post this as a PR comment and output locally after the review cycle completes. Adapt the template based on `reporting.verbosity` config.

## Full Report Template

```markdown
# PR Orchestrator Review Summary

**PR:** #{pr_number} — {pr_title}
**Repo:** {owner}/{repo}
**Reviewed:** {timestamp}
**Review Cycles:** {cycle_count}
**Autonomy Mode:** {autonomy_level}

---

## Executive Summary

{total_findings} findings from {tools_count} review tools. {applied_count} fixes applied, {skipped_count} skipped, {deferred_count} deferred. {new_issues_from_fixes} new issues introduced by fixes. CI status: {ci_status}.

---

## Review Tools

| Tool | Status | Findings | Time |
|------|--------|----------|------|
| {tool_name} | {ran/skipped/failed} | {count} | {duration} |
| ... | ... | ... | ... |

{tool_notes — e.g., "Jules was skipped due to budget constraints"}

---

## Findings

### Applied ({applied_count})

| # | Source | Severity | Category | File | Description | Commit |
|---|--------|----------|----------|------|-------------|--------|
| 1 | {tool} | {sev} | {cat} | {file}:{line} | {summary} | {sha} |
| ... |

### Skipped ({skipped_count})

| # | Source | Severity | Category | File | Description | Reason |
|---|--------|----------|----------|------|-------------|--------|
| 1 | {tool} | {sev} | {cat} | {file}:{line} | {summary} | {reason} |
| ... |

{Common skip reasons: "Not applicable to this codebase pattern", "False positive — code is correct as-is", "Out of PR scope — tracked in issue #XX", "Auto-skipped by learning system (tool precision below threshold)"}

### Deferred ({deferred_count})

| # | Source | Severity | File | Description | Tracked In |
|---|--------|----------|------|-------------|------------|
| 1 | {tool} | {sev} | {file}:{line} | {summary} | Issue #{num} |
| ... |

---

## Consensus Analysis

{If multiple tools flagged the same issue, highlight it here}

| Finding | Tools in Agreement | Confidence |
|---------|-------------------|------------|
| {description} | {tool1}, {tool2} | High |
| ... |

---

## Changes Made

| Commit | Files Changed | Finding(s) Addressed |
|--------|--------------|---------------------|
| {sha} | {files} | #{finding_ids} |
| ... |

---

## Test Results

| Suite | Status | Duration | Notes |
|-------|--------|----------|-------|
| {test_name} | PASS/FAIL | {time} | {notes} |
| ... |

{If tests failed after fixes, note which fix caused the failure and whether it was rolled back}

---

## Compliance

### Project Policies
- **claude.md**: {PASS/FLAG/N/A} — {summary}
- **agents.md**: {PASS/FLAG/N/A} — {summary}

### Regulatory Flags
{Only if legal_domains configured}

| Domain | Status | Details |
|--------|--------|---------|
| {domain} | {status} | {details} |

> These flags are automated awareness signals, not legal advice.

---

## Cost Summary

| Category | Value |
|----------|-------|
| Actions minutes used | {minutes} |
| API calls (GitHub) | {count} |
| API calls (AI services) | {count} |
| Estimated total cost | ${amount} |
| Budget remaining | ${amount} / ${budget} |

---

## Learning Insights

- **Tool performance this cycle:**
  - {tool}: {x}/{y} findings accepted ({precision}%)
- **Patterns observed:**
  - {e.g., "CodeRabbit continues to excel at TypeScript security findings"}
  - {e.g., "Jules style suggestions for Python had 0/3 accepted — consider disabling"}
- **Workflow optimization suggestions:**
  - {e.g., "Batching commits saved ~8 Actions minutes this cycle"}
  - {e.g., "Consider reducing max_iterations to 2 — cycle 3 has found 0 new issues in the last 5 PRs"}

---

*Generated by PR Orchestrator v{version} | [Configuration](.pr-orchestrator.yml) | [Learning Data](.pr-orchestrator/learning/)*
```

## Summary Report Template

For `reporting.verbosity: summary`:

```markdown
# PR Orchestrator Summary — #{pr_number}

**{applied_count} fixes applied, {skipped_count} skipped** from {total_findings} findings ({tools_count} tools).

| # | Decision | Source | Severity | File | Description |
|---|----------|--------|----------|------|-------------|
| 1 | Applied | {tool} | {sev} | {file} | {summary} |
| 2 | Skipped | {tool} | {sev} | {file} | {summary} — {reason} |
| ... |

CI: {status} | Cost: ${amount} | Cycles: {count}
```

## Minimal Report Template

For `reporting.verbosity: minimal`:

```markdown
**PR Orchestrator:** {applied_count} applied, {skipped_count} skipped, {deferred_count} deferred. CI {status}. Cost: ${amount}.
```
